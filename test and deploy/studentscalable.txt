Student Dashboard Pages & API Analysis
1. Student Dashboard Overview (dashboard_overview + get_dashboard_with_notifications)
API Calls:

student/dashboard/ (GET) - Basic dashboard data
student/dashboard-with-notifications/ (GET) - Combined dashboard + notifications
Backend Functions: dashboard_overview, get_dashboard_with_notifications
Current Implementation: Highly optimized with database aggregations, select_related, and limited queries
Issues/Improvements Needed:
✅ Good: Uses database aggregations (Count, Avg, Case/When) to avoid N+1 queries
✅ Good: select_related used for foreign key optimization
✅ Good: Limited leave requests to [:5] and notifications to [:15]
⚠️ Correlation Calculation: Complex correlation calculation done in Python - could be moved to database
⚠️ Multiple Queries: Separate queries for InternalMark and StudentIAMark - could be combined
⚠️ Add Caching: Implement Redis caching for dashboard data that doesn't change frequently
